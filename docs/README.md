# docs/ — 문서 인덱스

이 폴더에는 BanPickMod 프로젝트의 기획·설계·가이드 문서가 담겨 있습니다.  
코드를 작성하기 전에 반드시 아래 문서 순서대로 읽을 것을 권장합니다.

---

## 문서 목록 및 역할

| 파일 | 역할 | 우선순위 |
|---|---|---|
| `Plan_launcher_revised.md` | **핵심 기획서** — 전체 시스템 구조, 사용자 플로우, 각 구성 요소(런처·모드·맵·DB·옵저버·멀티) 상세 설계 | ★★★ 필독 |
| `Design.md` | **UI/UX 설계** — 외부 런처 UI, 인게임 밴픽 UI, HUD, 색상/폰트 가이드 | ★★★ 필독 |
| `BanPick Style Guide.md` | **밴픽 화면 스타일 가이드** — 레이아웃 레퍼런스, 색상/폰트 원칙, 옵저버 HUD 연결 방향 | ★★ 권장 |
| `Plan.md` | **초안 기획서** (레거시) — 수정 전 구조 참고용. 현재 구조와 다른 부분이 있음. 신규 개발에는 `Plan_launcher_revised.md` 우선 적용 | ★ 참고용 |

## 각 문서 사용 가이드

### `Plan_launcher_revised.md` — 읽는 순서

1. **§1 개요** — 목표, 사용자 플로우, 대상 환경
2. **§2 시스템 구조** — 최종 구조 결정, 구성 요소 역할, 패키지/맵 풀/DB 구조
3. **§3 사용자 플로우 상세** — 단계별 UX 흐름 (런처→잠금→밴픽→시작)
4. **§4 외부 런처 설계** — 런처 역할 범위, SC2 실행 방식, 파일/경로/업데이트 규칙
5. **§5–6 인게임 밴픽 및 섬멸전 맵 설계** — 맵 초기화, 잠금 구조, 밴 결과 적용
6. **§7 데이터 교환 설계** — Bank 미사용 원칙, 전역 구조체, Game Attribute
7. **§8 모드/데이터 정의** — 트리거 폴더 구조, 데이터 모듈, DB 영역
8. **§9 옵저버 지원** — GameHeart 스타일 HUD, 쉐도우 밴 공개 범위
9. **§10–11 멀티/동기화, 에러/예외** — 역할 슬롯, 연결 끊김, 에러 처리
10. **§12 개발 단계 및 우선순위** — 로드맵, 단계별 목표

### `Design.md` — 읽는 순서

1. **§1 외부 런처 UI** — 런처 화면 구성 (EXE 앱)
2. **§2 인게임 밴픽 UI** — BPMStage별 화면 전환, 옵션 화면, 풀스크린 밴픽 레이아웃
3. **§3 매치 시작 연출**
4. **§4 인게임 HUD** — 플레이어/옵저버 밴 HUD, 명령창 피드백
5. **§5 심판 전용 UI**
6. **§6 UI 에셋 정의**
7. **§7 색상 및 폰트 가이드**
