# 밴 후 비공개 자기 픽 시스템 스펙

> 이 문서는 기존 밴픽 규칙 위에 `밴 이후 그룹 유닛 잔여분을 각자 비공개로 확정하는 단계`를 추가하기 위한 상세 스펙이다.
> 기준 문서는 `Plan_launcher_revised.md`이며, 본 문서는 확장 유닛/그룹 유닛 처리 규칙만 따로 구체화한다.
>
> **핵심 원칙**: Hidden Self Pick 단계는 그룹(`GroupId`)이 지정된 유닛이 있는 경우에만 적용된다.
> 그룹에 속하지 않은 확장 유닛은 자동으로 픽에 포함되며, 별도 선택 행위가 필요 없다.
>
> **그룹 정의**: 그룹은 `UnitMeta.GroupId` 필드로 관리하며, DB 수정만으로 추후 변경 가능하다.

---

## 1. 목표

확장 유닛 풀이 커지면 밴 개수가 고정되어 있어도 일부 유닛은 반드시 남는다.
특히 `불곰-파이어뱃`처럼 같은 그룹으로 묶인 유닛은 그룹 보호 규칙 때문에 한 그룹 안에서 일부만 밴되고 나머지가 자동 생존할 수 있다.

이 문제를 해결하기 위해 밴 단계 뒤에 `비공개 자기 픽(Hidden Self Pick)` 단계를 추가한다.
이 단계는 **그룹 유닛이 있는 경우에만** 적용된다.

- 밴 단계에서는 상대가 못 쓰게 만들 유닛을 제한된 개수만큼 고른다.
- 픽 단계에서는 **그룹 유닛 중 밴되지 않고 남은 것**을 각자 비공개로 선택한다.
  - 그룹에 속하지 않은 확장 유닛은 밴만 피하면 자동으로 사용 가능하므로 따로 픽할 필요가 없다.
- 픽 결과는 상대에게 공개되지 않는다.

핵심 의도는 다음과 같다.

- 그룹 보호 규칙 때문에 억지로 남는 그룹 유닛을 강제 사용하지 않도록, 마지막에 선택권을 준다.
- 그룹이 없는 확장 유닛은 밴 단계 외에 추가 선택 부담을 주지 않는다.
- 기본 섬멸전 뼈대 유닛은 흔들지 않고, 확장 유닛만 전략적으로 구성하게 만든다.

---

## 2. 규칙 요약

예시: `5밴`

- 기본 유닛은 밴 불가다.
  - 일꾼
  - 각 종족 기본 전투 시작점 유닛
    - 테란: 해병
    - 프로토스: 광전사
    - 저그: 저글링
- 각 플레이어는 정해진 밴 수만큼 밴을 수행한다.
- 어떤 유닛이 그룹에 속해 있으면, 그 그룹에서 하나가 밴된 뒤에는 같은 그룹의 나머지 유닛은 추가 밴 불가가 된다.
- 밴이 끝난 뒤:
  - **그룹 유닛이 있으면**: 각 플레이어는 그룹 유닛 잔여분 중 자신이 사용할 것을 비공개로 선택한다.
  - **그룹 유닛이 없으면**: Hidden Self Pick 단계를 건너뛴다.
- 그룹에 속하지 않은 확장 유닛은 별도 픽 없이 자동으로 사용 가능하다.
- 상대는 내 픽 결과를 볼 수 없고, 나도 상대 픽 결과를 볼 수 없다.
- 최종적으로는 아래만 사용 가능하다.
  - 기본 유닛
  - 그룹에 속하지 않은 확장 유닛 중 밴되지 않은 것 (자동)
  - 그룹 유닛 중 밴되지 않았고 내가 픽한 것

---

## 3. 용어

- `기본 유닛(Base Units)`
  - 밴도 픽도 타지 않는 공통 보장 유닛.
  - 최소 기준은 `일꾼 + 해병/광전사/저글링`.
- `확장 유닛(Extended Units)`
  - 공허의 유산 기본 뼈대 밖에서 추가되는 모든 유닛.
  - 캠페인 유닛, 스타1 복각 유닛, 프로젝트 전용 추가 유닛을 포함한다.
- `그룹 유닛(Grouped Units)`
  - `UnitMeta.GroupId`가 지정된 유닛. 같은 `GroupId`를 공유하는 유닛끼리 하나의 그룹을 이룬다.
  - 밴 단계에서는 그룹 내 1개만 밴 가능하다.
  - 픽 단계에서는 그룹 내 남은 유닛을 각자 선택한다.
  - **그룹 정의는 DB(`UnitMeta.GroupId`)에서 관리하므로 코드 변경 없이 추후 수정 가능하다.**
- `비그룹 확장 유닛(Ungrouped Extended Units)`
  - `UnitMeta.GroupId`가 없는 확장 유닛.
  - 밴만 피하면 자동으로 사용 가능 상태가 된다. 별도 픽 선택이 필요 없다.
- `Hidden Self Pick`
  - 밴 종료 후, 각 플레이어가 **그룹 유닛 잔여분** 중 사용할 것을 비공개로 확정하는 단계.
  - 그룹 유닛이 전혀 없는 풀에서는 이 단계를 건너뛴다.

## 4. 단계별 플로우

### 4.1 옵션 확정

- 기존과 동일하게 `UnitPoolMode`, `BanCount`, 선공/후공, 공개/쉐도우 밴 규칙을 먼저 확정한다.
- `Hidden Self Pick`은 현재 유닛 풀에 그룹 유닛(`GroupId` 지정 유닛)이 하나라도 있으면 자동 활성화한다.
- 순수 공허 바닐라 규칙만 쓰는 모드(그룹 유닛 없음)에서는 자동으로 건너뛴다.

### 4.2 밴 단계

- 각 플레이어는 기존 규칙대로 정해진 개수만큼 밴한다.
- 기본 유닛은 후보에서 제외한다.
- 이미 밴된 유닛은 다시 선택할 수 없다.
- 그룹 유닛은 아래 규칙을 따른다.
  - 그룹에서 첫 밴이 발생하면 해당 그룹은 `ban-locked` 상태가 된다.
  - 같은 그룹의 나머지 유닛은 이번 밴픽 세션 동안 더 이상 밴 후보에 나타나지 않는다.
- 쉐도우 밴이 있는 경우에도 내부 판정은 일반 밴과 동일하게 처리한다.

### 4.3 비공개 자기 픽 단계

> 이 단계는 현재 풀에 그룹 유닛이 있을 때만 진행한다. 그룹 유닛이 없으면 건너뛴다.

- 밴 단계가 끝나면 각 플레이어는 동시에 자신의 `그룹 유닛 잔여 목록`을 본다.
- 이 목록에는 아래 조건을 모두 만족하는 유닛만 포함된다.
  - `UnitMeta.GroupId`가 지정됨 (그룹 유닛)
  - 현재 유닛 풀에 포함됨
  - 기본 유닛이 아님
  - 자신에게 적용된 밴 결과에 포함되지 않음
- 각 플레이어는 목록에서 `사용할 그룹 유닛`만 선택하고 확정한다.
- 확정 전까지는 자유롭게 선택/해제할 수 있다.
- 확정 후에는 상대에게 다음만 보인다.
  - `상대가 픽을 완료했다`는 상태
  - 실제 어떤 유닛을 골랐는지는 비공개
- **그룹에 속하지 않은 확장 유닛은 이 화면에 표시되지 않는다.** 밴만 피하면 자동 사용 가능.

### 4.4 경기 시작

- 기본 유닛은 항상 사용 가능하다.
- 그룹 유닛은 `밴되지 않았고 내가 픽한 것`만 사용 가능하다.
- 비그룹 확장 유닛은 밴되지 않은 것이면 모두 자동으로 사용 가능하다.

## 5. 후보 생성 규칙

### 5.1 밴 후보

밴 후보는 다음 조건을 모두 만족해야 한다.

- `UnitMeta.IsBanEligible == true`
- `UnitMeta.IsBaseUnit == false`
- 현재 `UnitPoolMode`에 포함됨
- 이미 밴되지 않음
- 그룹 보호 규칙에 의해 잠기지 않음

### 5.2 픽 후보 (Hidden Self Pick 대상)

픽 후보는 **그룹 유닛에만 적용**된다. 다음 조건을 모두 만족해야 한다.

- `UnitMeta.IsExtendedUnit == true`
- `UnitMeta.IsBaseUnit == false`
- `UnitMeta.GroupId != null` (그룹 유닛)
- 현재 `UnitPoolMode`에 포함됨
- 자신에게 적용된 밴 결과에 포함되지 않음

즉, 픽 후보는 `그룹 유닛 중 살아남은 것들`이다.

### 5.3 자동 포함 대상 (비그룹 확장 유닛)

아래 조건을 만족하는 유닛은 픽 단계 없이 자동으로 사용 가능 상태가 된다.

- `UnitMeta.IsExtendedUnit == true`
- `UnitMeta.IsBaseUnit == false`
- `UnitMeta.GroupId == null` (그룹 미지정)
- 현재 `UnitPoolMode`에 포함됨
- 자신에게 적용된 밴 결과에 포함되지 않음

이 유닛들은 `HiddenSelfPicks`가 아닌 `AutoIncludedUnits`로 별도 추적한다.

---

## 6. 그룹 유닛 처리

### 6.1 밴 단계

- 그룹 단위 강제 삭제가 아니라 `그룹 내 추가 밴 금지`가 핵심이다.
- 예시:
  - `불곰`, `파이어뱃`이 같은 그룹(`GroupId = "TerranHeavyInfantry"` 등)이라고 가정한다.
  - `불곰`이 밴되면 `파이어뱃`은 자동 밴되지 않는다.
  - 대신 같은 그룹에서 추가 밴은 막힌다.
- 그룹은 `UnitMeta.GroupId` 필드 값으로 판단한다. DB 수정만으로 그룹 구성을 바꿀 수 있다.

### 6.2 픽 단계

- 픽 단계에서는 그룹 보호 규칙을 그대로 가져오지 않는다.
- 남아 있는 그룹 유닛은 각각 **개별 픽 대상**으로 본다.
- 예시:
  - `불곰` 밴 후 `파이어뱃`이 남았으면, 플레이어는 `파이어뱃`을 쓸지 말지 직접 선택한다.
- **그룹에 속하지 않은 확장 유닛**은 픽 화면에 표시되지 않고 자동으로 사용 가능 처리된다.

이 규칙의 목적은 `그룹 때문에 남은 유닛`을 다시 플레이어 의사로 정리하게 만드는 것이다.
그룹에 속하지 않은 유닛은 이미 밴 단계에서 충분히 걸러지므로 추가 선택 단계를 넣지 않는다.

## 7. 최종 사용 가능 유닛 계산

각 플레이어의 최종 비활성화 목록은 아래처럼 계산한다.

```text
FinalDisabledUnits[player]
= AllBannedUnitsAgainstPlayer
+ GroupedUnitsNotPickedByPlayer
```

추가 조건:

- `BaseUnits`는 `FinalDisabledUnits`에 절대 포함하지 않는다.
- 밴된 유닛은 픽 여부와 무관하게 비활성화한다.
- 그룹 유닛 중 밴되지 않았더라도 픽하지 않은 것은 비활성화한다.
- **비그룹 확장 유닛 중 밴되지 않은 것은 자동으로 활성 상태**이므로 `FinalDisabledUnits`에 포함하지 않는다.

권장 내부 구조:

```text
BaseUnits[player]
GroupedExtendedUnits[player]        // GroupId 지정 확장 유닛
UngroupedExtendedUnits[player]      // GroupId 미지정 확장 유닛 (자동 포함)
PublicBansAgainst[player]
ShadowBansAgainst[player]
SelfBans[player]
HiddenSelfPicks[player]             // 그룹 유닛 픽 결과만
AutoIncludedUnits[player]           // 비그룹 확장 유닛 중 생존 목록
FinalDisabledUnits[player]
```

## 8. UI 요구사항

### 8.1 밴 화면

- 기존 밴 화면을 유지한다.
- 그룹 보호로 인해 더 이상 밴할 수 없는 유닛은 아래 중 하나로 표현한다.
  - 잠금 오버레이
  - `같은 그룹 유닛이 이미 밴됨` 툴팁

### 8.2 픽 화면

> 이 화면은 현재 풀에 그룹 유닛이 있을 때만 표시한다. 그룹 유닛이 없으면 생략한다.

- 밴 종료 후 별도의 풀스크린 픽 단계로 전환한다.
- 좌우 공개 밴 리스트는 유지해도 되지만, 중앙 그리드는 **그룹 유닛 잔여분만** 보여준다.
  - 비그룹 확장 유닛은 이 화면에 표시하지 않는다.
- 각 유닛은 `선택됨 / 선택 안 됨` 두 상태를 가진다.
- 하단 버튼:
  - `모두 해제`
  - `확정`
- 상대 영역에는 실제 픽 목록 대신 상태만 보여준다.
  - `선택 중`
  - `확정 완료`

### 8.3 정보 비공개

- 플레이어 화면:
  - 자신의 그룹 유닛 픽만 실제 아이콘/이름으로 표시
  - 비그룹 확장 유닛은 별도 표시 없이 자동 포함됨을 안내 문구로만 표시 가능
  - 상대 픽은 절대 표시하지 않음
- 옵저버/심판 화면:
  - 기본값은 양쪽 픽 비공개 유지
  - 토너먼트 모드에서는 양측 실제 픽 공개 옵션 추가 가능

## 9. 데이터 구조 변경안

`UnitMeta`에 아래 필드를 추가하는 것을 권장한다.

- `IsBaseUnit`
  - 밴/픽 모두 제외되는 기본 유닛 여부
- `IsExtendedUnit`
  - 확장 유닛 여부 (그룹 포함 비그룹 모두)
- `GroupId`
  - 그룹 유닛 판정용 ID. 값이 없으면(`null` 또는 빈 문자열) 비그룹 확장 유닛으로 처리한다.
  - **추후 그룹 구성을 변경하려면 이 필드만 수정하면 된다. 코드 변경 불필요.**
- `PickPriority`
  - 픽 화면 기본 정렬용 값

런타임 구조체 권장안:

```text
GroupBanLocked[groupId]                  // 그룹 잠금 상태 (플레이어별 아님, 그룹 전체 공유)
HiddenSelfPickCandidates[player][]       // 그룹 유닛 잔여분만
HiddenSelfPicks[player][]                // 확정된 그룹 유닛 픽
AutoIncludedUnits[player][]              // 비그룹 확장 유닛 중 생존한 것 (자동 포함)
PickConfirmed[player]
```

---

## 10. 동기화 규칙

- 밴 결과는 기존처럼 전 세션 공용 구조체에 즉시 반영한다.
- 그룹 유닛 픽 결과는 각 플레이어 로컬 입력으로 수집하되, 확정 시점에는 게임 세션 권위 구조체에 기록한다.
- 비그룹 확장 유닛의 생존 목록(`AutoIncludedUnits`)은 밴 결과 확정 시 자동으로 계산된다. 별도 동기화 입력이 필요 없다.
- 상대 클라이언트에는 `PickConfirmed[player]`만 노출한다.
- 경기 시작 직전 `FinalDisabledUnits[player]` 계산 시점에 그룹 유닛 픽과 비그룹 유닛 자동 포함을 합산한다.

## 11. 예시 시나리오

### 11.1 5밴 + 그룹 유닛 있음

- 기본 유닛:
  - SCV / Probe / Drone
  - Marine / Zealot / Zergling
- 확장 유닛:
  - Marauder (`GroupId = "HeavyInfantry"`)
  - Firebat (`GroupId = "HeavyInfantry"`)
  - Reaver (그룹 없음)
  - Medic (그룹 없음)
  - Goliath (그룹 없음)
- 그룹:
  - `Marauder`, `Firebat`은 같은 그룹 (`HeavyInfantry`)

진행 예:

1. 상대가 `Marauder`를 밴한다.
2. `Firebat`은 그룹 보호 때문에 추가 밴 불가 상태가 된다.
3. 나머지 밴이 모두 끝난다.
4. 픽 단계가 열린다. 그리드에는 그룹 유닛 잔여분(`Firebat`)만 표시된다.
   - `Reaver`, `Medic`, `Goliath`는 비그룹 유닛이므로 픽 화면에 표시되지 않고 자동 포함된다.
5. 나는 픽 단계에서 `Firebat`을 선택하거나 포기한다.
6. 최종 사용 가능 유닛:
   - 자동 포함: `Reaver`, `Medic`, `Goliath` (비그룹, 밴 안 당함)
   - 픽 결과에 따라: `Firebat` (픽했으면 사용 가능, 포기하면 비활성화)

---

## 12. 구현 시 주의점

- 기본 유닛과 확장 유닛의 경계를 DB에서 명확히 나눠야 한다.
- 그룹 유닛과 비그룹 유닛의 경계는 `UnitMeta.GroupId` 유무로만 판단한다. 로직에서 유닛 ID를 직접 비교하지 않는다.
- Random 종족은 밴 후보/픽 후보를 만들기 전에 내부 종족을 먼저 확정해야 한다.
- 픽 단계 제한 시간이 끝났을 때 정책이 필요하다. (그룹 유닛에만 해당)
  - 권장 1안: 현재 선택 상태 그대로 자동 확정
  - 권장 2안: 아무것도 선택하지 않았으면 `모든 잔여 그룹 유닛 선택`으로 자동 보정
- 밴 적용, 그룹 유닛 픽 미선택 적용, 비그룹 유닛 자동 포함을 같은 `FinalDisabledUnits` 파이프라인으로 합쳐야 유지보수가 쉽다.
- 그룹 구성을 나중에 변경할 때는 `UnitMeta.GroupId` 값만 수정하면 된다. 밴/픽 로직 코드는 건드리지 않아도 된다.

## 13. 현재 가정

이 스펙은 아래 가정을 기준으로 작성했다.

- Hidden Self Pick은 **그룹 유닛(`GroupId` 지정)에만** 적용된다.
- 비그룹 확장 유닛은 밴만 피하면 자동으로 사용 가능하며 픽 선택이 필요 없다.
- 기본 유닛은 항상 사용 가능하다.
- 그룹 규칙은 `밴 단계`에만 강제되고 `픽 단계`에는 강제되지 않는다.
- 상대 픽은 플레이어에게 비공개이며, 옵저버 공개 여부는 별도 옵션으로 둔다.
- 그룹 구성(`GroupId` 값)은 추후 결정하며 DB 수정만으로 변경 가능하다.

이 가정이 실제 의도와 다르면 우선 아래를 다음 문서 업데이트에서 확정하면 된다.

- `그룹 유닛도 픽 시 그룹 단위로 강제 선택` (현재는 그룹 내 개별 픽 허용)
- `공허 바닐라 유닛 일부도 픽 대상으로 확장` (현재는 확장 유닛 + 그룹 유닛만 해당)
