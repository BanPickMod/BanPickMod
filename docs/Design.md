# 프로젝트 디자인 (UI/UX) 요구사항 정리

> 이 문서는 `Plan_launcher_revised.md`의 시스템 구조와 `BanPick Style Guide.md`를 기반으로 작성된 통합 UI/UX 설계 문서입니다.  
> 외부 런처(EXE), BPM 전용 섬멸전 맵 내 인게임 UI, 인게임 HUD를 모두 포함합니다.

---

## 1. 외부 런처 UI (`BPM Launcher.exe`)

> 런처는 설치/업데이트/실행 도구이며, 게임 룰 판정 주체가 아닙니다.  
> SC2 내부 UI가 아닌 Windows 애플리케이션입니다.

### 1.1 메인 화면

- **상태 라벨**: "설치 안 됨" / "설치됨" / "업데이트 가능" / "오류" 등 현재 상태를 명확히 표시.
- **액션 버튼 (4가지)**:
  - 설치 / 업데이트 / 파일 확인 / 제거
- **실행 버튼 ("Play")**: 설치 완료 시 활성화되는 크고 직관적인 버튼.
- **진행 상태 바**: 패키지 다운로드/압축 해제 진행률 표시.

### 1.2 맵 선택 (로컬 테스트)

- 설치된 `BPM_1v1_*.SC2Map` 목록을 드롭다운 또는 리스트로 표시.
- 기본값: `BPM_1v1_TestMap.SC2Map`.

### 1.3 고급 설정 화면

- SC2 설치 경로 수동 선택 (폴더 찾아보기 버튼).
- 캐시 삭제, 사용자 설정 초기화 버튼.
- 패치노트/버전 정보 뷰어.

### 1.4 에러 및 예외 다이얼로그

| 오류 상황 | 표시 내용 |
|---|---|
| SC2 경로 없음/잘못됨 | "StarCraft II 설치 경로를 찾을 수 없습니다." + 경로 선택 UI |
| 필수 모드/맵 파일 누락 | 누락 파일 목록 + 설치/업데이트 버튼 |
| 다운로드/검증 실패 | 오류 안내 + 캐시 삭제 권장 + 재다운로드 버튼 |
| SC2 비정상 종료 | 간단한 원인/안내 메시지 |

### 1.5 런처 ↔ SC2 데이터 교환 범위

- 런처가 넘길 수 있는 정보: 로컬 실행 맵 경로, UI 언어/설정, 비권위 메타 정보(외부 매치 ID 등).
- **넘기지 않는 정보**: 밴픽 결과, 금지 유닛 목록, 승패 판정 — 이는 SC2 맵 내부에서 결정.

## 2. 인게임 밴픽 UI (BPM 전용 섬멸전 맵 내부)

> 게임 시작 직후 3, 2, 1 카운트다운 이후, 플레이어 입력을 잠그고 전체 화면으로 표시하는 밴픽 전용 UI입니다.  
> LoL 챔피언 선택(그리드 중심 + 상단 진행 정보)을 베이스로, 도타2식 역할 필터를 부분 적용합니다.

### 2.1 화면 전환 및 배경 처리

- **배경**: 전체 화면을 덮는 어두운 반투명 `BGPanel` 처리.
- **단계(`BPMStage`)에 따른 UI 전환**:
  1. `0` 초기 잠금 → 옵션/Ready 화면 표시
  2. `1` 옵션 확인 → 룰 설정 화면
  3. `2` 밴픽 진행 → 풀스크린 밴픽 UI
  4. `3` 밴 결과 적용 → "Loading Match..." 연출
  5. `4` 매치 시작 연출
  6. `5` 섬멸전 진행 중

### 2.2 옵션/Ready 화면 (BPMStage 1)

- **설정 항목 및 UI 컴포넌트**:

  | 설정 | UI 형식 | 값 |
  |---|---|---|
  | 유닛 풀 버전 | 토글 버튼 또는 드롭다운 | 공허 바닐라 / 캠페인+스타1 |
  | 밴 개수 | 버튼 그룹 또는 슬라이더 | 0 / 1 / 3 / 5 |
  | 선공 설정 | 버튼 그룹 | Player 1 / Player 2 / Random |
  | 쉐도우 밴 | 토글 버튼 | 사용 / 미사용 |

- **권한 제어**:
  - 심판이 있으면 심판만 확정 버튼 활성화.
  - 심판이 없으면 HostPlayer(기본: Player 1)가 확정 권한.
  - PlayerA/B: Ready 또는 동의 버튼 표시.
  - 옵저버: 읽기 전용 (버튼 비활성화 또는 커서 변경으로 표현).

- **동적 텍스트 갱신**: 옵션 선택 시 "예상 밴 라운드 수" 및 설명 텍스트를 즉시 갱신.
- **툴팁**: 각 옵션 항목 위에 마우스 오버 시 간단한 설명 표시.

- **플레이어 슬롯 레이아웃**:
  - 좌측: Player 1, Player 2 슬롯 (이름, 종족, 색상, Ready 체크박스/버튼).
  - 우측: 옵저버 슬롯 목록 (스크롤바 포함, 옵저버 수 증가 대응).
  - 상단: 로고/제목, 플레이어 이름, 안내 텍스트.
  - 하단: 상태 메시지, "밴픽 시작" 버튼 (권한 및 조건에 따른 비활성/활성).

- **역할 표시 태그**: 플레이어 슬롯에 "Host", "Referee" 뱃지(라벨) 부착.

### 2.3 풀스크린 밴픽 UI (BPMStage 2)

#### 상단 바 (진행 정보)

- **좌측**: Player 1 이름, 종족 아이콘, 팀 색상.
- **중앙**: 현재 단계 텍스트 (예: `BAN 1`, `BAN 2`), 큰 숫자 타이머 + 얇은 진행 바.
- **우측**: Player 2 이름, 종족 아이콘, 팀 색상 (대칭 구조).
- **타이머 강조**: 10초 이하로 진입 시 카운트다운 숫자 및 진행 바를 경고색으로 강조.

#### 좌/우 밴 리스트

- **좌측 세로 스택**: 내 팀이 수행한 밴 리스트.
- **우측 세로 스택**: 상대에게 적용된 밴 리스트.
- **아이콘 표시 규칙**:
  - 공개 밴: 실제 유닛 아이콘 + 팀 컬러 테두리.
  - 쉐도우 밴: 실루엣/물음표 아이콘 → 게임 시작 시 실제 아이콘으로 전환.
  - 자기 밴: 유닛 아이콘 + 붉은 X 오버레이.

#### 중앙 유닛 그리드

- **크기**: 6×N ~ 8×N 아이콘 그리드.
- **각 아이콘**: 유닛 초상화, 종족/역할 색상 테두리, 밴 여부 오버레이.
- **필터/검색 바** (그리드 상단):
  - 종족 필터, 역할 필터 (탱커 / 근접 딜러 / 원거리 딜러 / 캐스터 / 지원 / 공성).
  - 텍스트 검색 입력.
- **비활성 처리**: 이미 밴된 유닛, 밴 불가 유닛(일꾼 등), 현재 풀에 없는 유닛은 흑백/회색조 처리 + 클릭 불가.
- **데이터 소스**: `UnitMeta` DB의 `IsBanEligible`, `Race`, `TechTier`, `Role`, `DefaultPool_A/B` 필드를 기반으로 그리드 구성 및 필터링.

#### 하단 정보/확인 영역

- **좌측**: 선택된 유닛의 큰 초상화, 이름, 종족/역할 아이콘, 한 줄 요약.
- **중앙**: "선택 / BAN 확정" 버튼 (내 차례일 때만 활성화), 선택적으로 "랜덤" 버튼.
- **우측 (간단 추천)**: 현재 조합/상대 종족 대비 추천 역할 아이콘 1~2개 + 짧은 텍스트 (예: "vs Mech에 강한 유닛").

#### 권한 제어

- 현재 라운드 Owner만 유닛 선택/확정 가능.
- 상대 플레이어/옵저버/심판: 관전 및 마우스 오버 툴팁만 허용, 확정 입력 불가.
- 옵저버 화면에 "관전자 - Player1 vs Player2" 라벨 표시.

## 3. 매치 시작 연출 (BPMStage 3~4)

> 이 단계는 실제 맵 로드가 아닙니다. 같은 `.SC2Map` 세션 안에서 밴픽 UI를 닫고 섬멸전 상태로 전환합니다.

- 밴픽 UI 종료.
- 화면 중앙에 "Loading Match..." 또는 "경기 시작 준비 중" 텍스트 렌더링.
- 자체 3, 2, 1 카운트다운 표시 후 잠금 해제 피드백.
- 시작 위치에 본진/일꾼 생성.
- 초기 자원/시야/카메라 설정.
- 플레이어 입력 잠금 해제.

## 4. 인게임 HUD (BPMStage 5 - 섬멸전 진행 중)

### 4.1 플레이어용 밴 HUD

- **위치**: 화면 상단 중앙.
- **ME / ENEMY 토글**: 단축키(예: `Alt+B`)로 내 밴 / 상대 밴 전환.
- **아이콘 슬롯**: 6~8개의 컴팩트한 슬롯 배치.
- **밴 종류에 따른 아이콘 렌더링 규칙**:
  - 공개 밴: 실제 유닛 아이콘.
  - 쉐도우 밴(상대에게 숨겨진 경우): 자물쇠/물음표 형태의 마스킹 아이콘.
  - 자기 밴: 유닛 아이콘 + 붉은 X 오버레이.

### 4.2 옵저버용 밴 HUD (GameHeart 스타일 확장)

- **위치**: 스코어보드 하단, 좌우 대칭형 밴 바(Ban Bar).
- **좌측 밴 바**: Player 1 관점, 파란색 계열 테두리.
- **우측 밴 바**: Player 2 관점, 빨간색/주황색 계열 테두리.
- **아이콘 슬롯**: 최대 N개 슬롯 (기본 5개).
- **마우스 오버 툴팁**: 유닛 이름 + 짧은 역할 설명.
- **쉐도우 밴 공개 범위** (설정으로 전환 가능):
  - **토너먼트 모드(기본값)**: 옵저버에게 쉐도우 밴의 실제 유닛도 공개.
  - **플레이어 시점 모드**: 쉐도우 밴을 마스킹 아이콘으로 숨긴 상태 유지.
- **시점 토글** (선택 사항):
  - `Ctrl+B`: 밴 HUD 전체 On/Off.
  - `Ctrl+1` / `Ctrl+2`: Player 1/2 관점으로 밴 바 강조.

### 4.3 인게임 명령창 피드백 (Ban 적용 시각화)

- 밴 당한 유닛의 생산 버튼, 진화/변태 버튼, 연구 업그레이드, 캠페인 패널, 용병 소환 버튼에 **자물쇠(Lock) 오버레이 또는 회색조 처리** 적용.
- 클릭 시도 시 "밴으로 인해 사용할 수 없습니다." 피드백 메시지 출력.

## 5. 심판(Referee) 전용 UI

- **역할 표시 뱃지**: 대기실 슬롯 또는 상단에 "Referee" 라벨 부착.
- **심판 전용 제어 패널** (심판 화면에서만 활성화):
  - 옵션 확정, 밴픽 시작, 밴픽 재설정, 진행 중단 버튼.
  - PlayerA/B Ready 상태 확인.
  - (후순위) 재경기 요청, 외부 결과 기록 버튼.

---

## 6. UI 데이터 에셋 정의 (`Data/UI`)

- **레이아웃/Frame**: 대기실, 맵 선택, 밴픽 풀스크린, 인게임 HUD 프레임 정의.
- **밴 아이콘 에셋**:
  - 잠금(Lock) 형태 아이콘: 공개 밴 HUD 슬롯에 활용.
  - 쉐도우(블라인드) 형태 아이콘: 쉐도우 밴 슬롯에 활용.
  - 붉은 X 오버레이: 자기 밴 시각화.
- **특수 옵션 아이콘**: 선공/후공 설정, 쉐도우 밴 등의 커스텀 아이콘.
- **명령창 피드백 에셋**: Ban Requirement 적용 시 비활성화 오버레이 (회색조, 자물쇠).

## 7. 색상 및 폰트 가이드

### 7.1 색상

| 구분 | 색상 |
|---|---|
| 배경 | 아주 어두운 남색 / 메탈릭 그레이 (SC2 기본 톤) |
| Player 1 팀 컬러 | 파란색 계열 (GameHeart 관례) |
| Player 2 팀 컬러 | 빨간색 / 주황색 계열 (GameHeart 관례) |
| 공개 밴 테두리 | 소속 팀 컬러 |
| 쉐도우 밴 테두리 | 회색 / 보라색 + 반투명 마스킹 |
| 자기 밴 오버레이 | 붉은색 X |
| 타이머 경고 | 붉은색 또는 황색 강조 |

### 7.2 폰트

- **크게/두껍게**: 타이머 숫자, 라운드 정보, 플레이어 이름 → 가독성 최우선.
- **얇게/작게**: 유닛 설명, 추천 텍스트, 툴팁 부가 정보 → 시선 분산 최소화.
